Details The vines in our project grow in relation to an L-System that we have implemented. L-Systems are used in computing to describes the behavior of plant cells and to model the growth of plants such as trees, see weed and vines. Our implementation of the L-System was based of the original system described by Lindenmayer (The author of the L-System). He used this algorithm to model the growth of Algae. In our project, we have done some small modification over the algorithm to create a more modular experience. Since our program is made to create vines in real time, we wanted the user to be able to create vines that corresponds to what he/she wants, so we added a variable that could control the amount of branching the L-System would have. With a high branching factor, the system would replicate the original L-System, but with a lower value, the number of branching would be smaller and the decision for when to branch would be random. With the minimum value, the vine would grow in a single branch and would never split.
Direction Variance
The direction in which new segments point.
Displays a low direction variance.
Displays a high direction variance.
Details In our project, the vines can be planted anywhere. Because of this, the vines needed to be able to detect the other objects in the scene to grow toward them and attach itself. So we added a bending and rotating value. Our code fetches the position of the nearest mesh and depending on its location, it will influence our vine to grow. The bending value can be change by the user, but depending on the location it will shift the value to either a positive one or a negative one, not influencing the intensity of the bending. The rotator works in an other manner. The user inputs a rotator value, this value defines how much the next branches to grow can have a rotation different from the parent, so if the parent is growing straight up (with a rotation of 0) and the user inputs a rotation value of 25, then the next part of the vine can grow in a direction ranging from -25 to 25. going back to our mesh mentioned previously, we have created a bias that would affect the rotation. The idea behind is that if the root of the vine is far from the nearest mesh, our bias will affect the rotation value to influence the way the vine will grow. The closer the vine is from its target mesh, the less it will be affected by the bias (and so, the more random it will grow)
Object Interaction

The interaction of plants and cubes.
The interaction of plants and an object.
Details To give a more realistic animation, the vines are implemented as a doubly linked list of branches. Every time a branch is colliding with an object a rotation is applied to it to get out of the collision, and when it doesn’t collide, it bends down because of the physic force given by the weight of the branches on the tree. The rotation of the branches is calculated at every frame so that the rotation looks smooth. When a rotation is applied to a child, this child registers its rotation and gives a diminished rotation value to its previous three parents (because of the physic force given by the weight of the branches on the previous branches). The position of the child pivot is recalculated according to the new position of its parent so that they are kept together.
